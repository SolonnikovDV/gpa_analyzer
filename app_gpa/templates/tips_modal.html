{# Модальное окно «Подсказки» для страницы детального анализа #}
Подсказки по работе с приложением
Режимы анализа
  • Только логика — парсер извлекает блоки и объекты из DDL, планы берутся из БД (EXPLAIN). Требуется подключение к Greenplum.
  • Гибрид — с БД: парсер + EXPLAIN; без БД: автоматически чистый агент. Нужен ключ GigaChat (в гибриде без БД).
  • Чистый агент — без БД: агент сам извлекает блоки и объекты, синтезирует планы. Поддерживает PL/pgSQL функцию, один SQL-запрос или скрипт (DDL/DML). Нужен ключ GigaChat.
Ключ GigaChat (для гибрида и чистого агента)
  • Нажмите «Ввести ключ» и вставьте токен из личного кабинета GigaChat. Или задайте в .env: GIGACHAT_CREDENTIALS=...
  • Профиль «разработчик» — токен из файла .key в корне проекта.
Что можно анализировать
  • PL/pgSQL функция — полный текст CREATE OR REPLACE FUNCTION ... AS $function$ ... $function$.
  • Один SQL-запрос — SELECT, INSERT, UPDATE, DELETE.
  • Скрипт — несколько DDL/DML (DROP TABLE, CREATE TABLE, INSERT и т.д.). Агент сам определит тип ввода.
  • Генерация по описанию — в гибриде: опишите запрос на естественном языке → «Сгенерировать запрос».
Подключение к БД
  • Опционально. Если включено — статистика таблиц подтягивается из Greenplum. Без БД — размеры задаются вручную на этапе настройки.
Настройка размеров и расчёт
  • Параметры функции — в том же порядке, что в сигнатуре. Строки в кавычках: '2024-01-01', 'current_date', 5.
  • Желаемое количество строк — задайте для оценки нагрузки при росте данных.
Антипаттерны и риск
  • Приложение детектирует тяжёлые паттерны (LATERAL с ORDER BY + LIMIT 1, CROSS JOIN, коррелированные подзапросы и др.) и добавляет множитель к нагрузке блока.
  • В сводке: Нагрузка (+X GB антипаттерны) — дополнительная оценка на скейле данных.
Сброс к базовым

Кнопка «Сброс к базовым» — сброс векторной базы, кэша ответов агента и состояний discovery.
